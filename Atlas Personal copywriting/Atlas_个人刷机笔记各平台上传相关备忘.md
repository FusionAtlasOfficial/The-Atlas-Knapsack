# 目前以及发现的网址针对性问题：
## 吾爱破解：
> 目前文章内部的基于 Markdown 语法生成的目录，以及文章内的直达链接在 52pojie 无法使用。
> * 我花了一早上使用论坛编辑器自带的 `“index”语法` 构建目录，但是因为目录太长，且默认显示在置顶，这样就导致每一页开头都被目录的内容大量占用，非常影响阅读和查阅。
>   * 更麻烦的是，使用 `“index”语法` 构建目录，必须要分页，而分页之后的 Markdown 排版会错乱，哪怕每一页都单独手动调整输入转换 Markdown 语法也一样，目前还暂时不知道要怎么处理这个问题...
>   * **注：页面左侧作者栏下方生成的目录可正常点击跳转。**
> * 论坛的 Markdown 发帖插件对 **“多级列表”** 仅支持有点兼容性问题，2层以上嵌套可能显示错误，而我在文章中为了展示上下文关系，是有一些3层甚至4层嵌套的，这可能会部分影响文章的可读性，目前已手动调整所有排版，如有纰漏欢迎指出。

吾爱破解目录页特殊说明：

> 因为 52pojie 论坛编辑器与 Markdown 语法格式仅能部分兼容，导致本页链接点击后无法直达跳转，但这个完全展开的目录起码展示了文章结构，便于查阅，所以依旧保留了，你可以使用页面左侧作者栏下方生成的目录，该目录可正常点击。
>
> 受此影响，后文正文中，文章内互相跳转的链接也无法实现点击直达跳转 **（外链不影响，可正常点击跳转）** 。
