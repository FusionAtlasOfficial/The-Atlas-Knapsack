# 关于上传平台

各大网站对于 Markdown 格式的支持以及标准都有所差异，首发排版可能有一些小问题，慢慢修。

本文同步发布于：

* Github 个人仓库：[【The-Atlas-Knapsack】](https://github.com/FusionAtlasOfficial/The-Atlas-Knapsack)
  * 因为本文注定是一篇会上传在多个平台的项目，但不同平台的政策和偏好完全不同，不见得每个平台上都能上传完整内容，所以 Github 上的版本算是原始存档。

* 吾爱破解：
  * 52pojie算是我非常喜欢的老牌论坛，但其实吧，毕竟实力有限，很多大佬的高端操作看不太懂，只能常年泡在悬赏区板块拾人牙慧，偶尔回答一点不是那么高深的问题，所以文章还没写完就在想到时候一定要发在52pojie，我确信这偏文章在这是有用武之地的。
    * 目前已经发现了文章内部的使用 Markdown 格式生成的目录，以及文章内的直达链接在52pojie无法使用，我花了一早上使用论坛编辑器自带的“index”语法构建目录，但是因为目录太长，且默认显示在置顶，这样就导致每一页开头都被目录的内容大量占用，非常影响阅读和查阅；
    * 更麻烦的是，使用“index”语法构建目录，必须要分页，而分页之后的 Markdown 排版必然会错乱，哪怕手动跳转每一页都是单独的md语法也一样，目前还不知道要怎么处理这个问题...暂时放弃了。
